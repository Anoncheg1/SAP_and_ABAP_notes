Дарья, добрый день! Для того, чтобы все у Вас было грамотно составлено, нужно не к риэлторам (маклерам) обращаться, а к юристам. Так как риэлторы даже не знают, что такое договор аренды, а что такое договор найма. Если Вы - физические лица, то договор Ваш называется НЕ договором аренды, а договором НАЙМА жилого помещения. Договор АРЕНДЫ в отношении ЖИЛОГО помещения заключается только с ЮРИДИЧЕСКИМИ лицами. С физическими - договор НАЙМА. Но если вы даже и "обзовете" свой договор договором аренды, то применяются к нему нормы Гражданского кодекса о договоре найма жилого помещения.

Договор найма жилого помещения заключается в простой письменной форме, на срок до пяти лет. Нотариальная форма - по желанию.

Если срок в договоре не указан, считается, что договор заключен на срок пять лет.

Никакой государственной регистрации такой договор НЕ подлежит (маклеры, путая аренду с наймом, часто говорят, что нужно заключить договор на срок до года, чтобы он не подлежал гос. регистрации).

Передача денег за ЖКУ и за наем (по договору) обязательно оформляется расписками.

В договоре обязательно напишите, какие ценные вещи оставляет в жилом помещении наймодатель, чтобы потом, не дай бог, не было претензий, что какие-то вещи пропали..

Стажер. Человек с любым набором знаний и навыков за спиной. Бывшие программисты быстрее вникают (мое наблюдение). Бывшие 1С-ники хуже — надо голову сломать. стажер, это человек, который ничего не знает про САП про тот модуль, на который он пришел. Он мог читать про систему, листать курсы, но это все. Обычно стажеры проходят обучение в компании, потом становятся «разнорабочими» на проектах. Бегают подписывать документы, собирают протоколы, выверяют данные при миграции, помогают с обучением пользователей. Как правило никогда не выполняют настроек за исключением галочек по базам для видов оплаты или настройки графиков рабочего времени.

Младший консультант. Бывший стажер, который прошел 1 проект полного цикла. Уже знает один-два компонента на уровне продвинутого пользователя. Может сам проводить обучение пользователей вместе со стажерами. Обычно такими становятся через год стажерства. Может сам провести миграцию по заданной методике. Настраивает простые вещи в системе, пишет базовые документы (рыбу проектного решения, сценарии тестирования, спецификации и пр.).

Консультант. Тот самый боец, про которого всегда говорят, что это саповские консы. Эти ребята уже прошли в среднем два проекта полного цикла. Могут сами закрывать небольшие запросы на изменения. Умеют работать с пользователями, понимают взаимосвязи внутри модуля и рядом. Могут с нуля настроить простой проект, написать простой план проекта, сделать базовые документы.

Старший консультант. Обычно это мозг, руководитель группы, который прошел 4-5 проектов и понимает модуль и «свои компоненты» на 4+. Такие люди могут запустить проект в два раза быстрее, чем младшие, например. Ибо уже есть опыт, понимание проблемных мест и наработанные инструменты и документы. 4-5 проектов обычно означает 3-5 лет живого опыта работы на проектах, отличных от стандарта системы. Вот эти люди уже могут претендовать на 200, если это 5 лет и 5 проектов со сложностями.

Ведущий консультант/Эксперт. Для меня ведущий и эксперт это одно и тоже. Эти ребята в сапе уже более 7-8 лет. Проекты прошли разные. Обычно выполняют роль архитекторов, руководителей проектов или фрилансят на сложных задачах. Они сразу могут дать ответы на 90% вопросов (а не на все, как думают некоторые).

Теперь отвечаю на вопрос, что стажер может получить после обучения у меня. Индивидуальное обучение позволяет понять, но не изучить работу системы. То есть на выходе будет понимание как настроить IDES или копию 000 манданта с нуля до базовых процессов (простой расчетный лист). Это не даст существенную прибавку к заработной плате, но позволит быстрее реализовать себя на проекте. Важно именно это, само время, которое вы экономите после обучения у меня. У меня есть примеры, когда студенты «отбивали» свои затраты через 1 месяц. Как MBA в свое время ??

CL_SALV_TREE
CL_SALV_TREE is a kind of wrapper class for CL_GUI_ALV_TREE, that is all the tree functionality in CL_SALV_TREE is actually implemented using CL_GUI_ALV_TREE. CL_SALV_TREE architeturally is more Object Oriented, if I can put it this way, for example, if you need some manipulating with display layout, then in CL_GUI_ALV_TREE you set different fields of special structure whilst in case of CL_SALV_TREE you first have to obtain a reference to a specific interface, then change layout by calling different methods of this interface. Thus, to some extent the source code with CL_SALV_TREE becomes less ABAP specific and can look similar to Java code. If you need only the default values in display layout and field catalogue, then the source code is expected to be very short with CL_SALV_TREE, otherwise the code grows very quickly if you need more control and more bells-n-whistles.

Batch input is a automatic procedure referred to as BDC(Batch Data Communications).The central component of the transfer is a queue file which receives the data vie a batch input programs and groups associated data into "sessions".
